### 4.2 核心表结构

#### 4.2.1 组织架构表 (sys_org)

```sql
CREATE TABLE sys_org (
    id              BIGSERIAL PRIMARY KEY,
    parent_id       BIGINT,                    -- 父级ID
    org_code        VARCHAR(50) UNIQUE,        -- 组织编码
    org_name        VARCHAR(200),              -- 组织名称
    org_type        VARCHAR(20),               -- 组织类型
    sort_order      INT DEFAULT 0,             -- 排序
    leader_name     VARCHAR(100),              -- 负责人
    contact_phone   VARCHAR(20),               -- 联系电话
    status          VARCHAR(10) DEFAULT '1',   -- 状态
    create_time     TIMESTAMP DEFAULT NOW(),
    update_time     TIMESTAMP DEFAULT NOW()
);
```

#### 4.2.2 用户表 (sys_user)

```sql
CREATE TABLE sys_user (
    id              BIGSERIAL PRIMARY KEY,
    org_id          BIGINT REFERENCES sys_org(id),

    password        VARCHAR(255),
    real_name       VARCHAR(100),
    phone           VARCHAR(20),
    email           VARCHAR(100),
    status          VARCHAR(10) DEFAULT '1',

```

## 5. API接口规范

### 5.1 RESTful API设计原则

- 使用HTTP动词表示操作：GET(查询)、POST(创建)、PUT(更新)、DELETE(删除)
- 资源命名使用复数名词
- 使用HTTP状态码表示结果
- 统一响应格式

### 5.2 统一响应格式

```json
{
    "code": 200,                    // 业务状态码
    "message": "操作成功",           // 消息描述
    "data": { },                    // 业务数据
    "timestamp": 1711612800000,     // 时间戳
    "requestId": "uuid",            // 请求追踪ID
    "page": {                       // 分页信息(可选)
        "current": 1,
        "size": 10,
        "total": 100,
        "pages": 10
    }
}
```

### 5.3 业务状态码定义

| 状态码 | 说明 |
|--------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未认证 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

### 5.4 核心API接口示例

#### 采购计划接口

```
GET    /api/procurement/plan/v1/plans                    # 查询计划列表
POST   /api/procurement/plan/v1/plans                    # 创建计划
```

## 6. 安全设计

### 6.1 认证授权架构

```
┌──────────────────────────────────────────────────────────────────┐
│                        认证授权流程                                │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│   用户请求 ──► 网关验证Token ──► 用户服务验证 ──► 权限服务鉴权     │
│                      │                   │              │       │
│                      ▼                   ▼              ▼       │
│                 JWT解析           用户信息获取      权限校验      │
│                                      │              │          │
│                                      ▼              ▼          │
│                               Redis会话       权限策略          │
│                               缓存验证        执行               │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

### 6.2 JWT认证方案

#### 6.2.1 Token结构

```json
{
    "header": {
        "alg": "HS256",
        "typ": "JWT"
    },
    "payload": {
        "sub": "user_id",
        "username": "admin",
        "orgId": "org_001",
        "roles": ["ROLE_ADMIN", "ROLE_USER"],
        "permissions": ["plan:read", "plan:write", "requirement:read"],
        "iat": 1711612800,
        "exp": 1711616400
    }
}
```

#### 6.2.2 Token生命周期

| 类型 | 有效期 | 说明 |
|------|--------|------|
| AccessToken | 1小时 | 访问令牌 |
| RefreshToken | 7天 | 刷新令牌 |
| 验证码Token | 5分钟 | 验证码校验 |

### 6.3 RBAC权限模型

#### 6.3.1 权限层次结构

```
角色 (Role)
    │
    ├── 功能权限 (Function Permission)
    │       ├── 菜单权限
    │       ├── 按钮权限
    │       └── API权限
    │
    └── 数据权限 (Data Permission)
            ├── 组织权限
            ├── 部门权限
            └── 项目权限
```

#### 6.3.2 权限控制粒度

| 层级 | 控制内容 | 实现方式 |
|------|----------|----------|
| 接口级 | API访问权限 | Spring Security Method |
| 菜单级 | 功能可见性 | 前端路由守卫 |
| 按钮级 | 操作按钮 | 自定义指令 |
| 数据级 | 数据可见范围 | MyBatis拦截器 |
| 字段级 | 字段读写 | 动态脱敏 |

#### 6.3.3 预定义角色

| 角色编码 | 角色名称 | 权限范围 |
|----------|----------|----------|
| ROLE_SUPER_ADMIN | 超级管理员 | 系统全部权限 |
| ROLE_ORG_ADMIN | 单位管理员 | 本单位全部权限 |
| ROLE_PLAN_ADMIN | 计划管理员 | 计划模块全部权限 |
| ROLE_REQUIREMENT_ADMIN | 需求管理员 | 需求模块全部权限 |
| ROLE_DOCUMENT_ADMIN | 文件管理员 | 文件模块全部权限 |
| ROLE_REVIEW_ADMIN | 评审管理员 | 评审模块全部权限 |
| ROLE_CONTRACT_ADMIN | 合同管理员 | 合同模块全部权限 |
| ROLE_AUDITOR | 审计员 | 只读权限 |
| ROLE_USER | 普通用户 | 业务操作权限 |

### 6.4 安全防护措施

| 防护类型 | 措施 | 说明 |
|----------|------|------|
| 传输安全 | HTTPS | 全链路TLS加密 |
| 接口防护 | 限流/防爬 | 网关限流策略 |
| SQL注入 | 参数绑定 | MyBatis预编译 |
| XSS攻击 | 输入过滤 | 富文本转义 |
| CSRF | Token验证 | 请求令牌 |
| 敏感数据 | 加密存储 | AES/RSA加密 |
| 日志审计 | 操作记录 | 全链路日志追踪 |
