## 1. 系统概述

### 1.1 项目背景

智能化采购系统旨在实现采购全流程的线上化、模板化管理，具备计划执行过程的自动预警、进度跟踪与双向考核功能。系统通过AI技术提升采购效率，实现智能辅助编制、智能合规检查、智能审核等功能。

### 1.2 系统范围

系统覆盖采购全生命周期，包括：
- 采购计划管理（年度计划、季度计划）
- 采购需求管理
- 采购文件管理与评审
- 采购合同管理与履约跟踪
- 档案管理与数据查询
- 质疑投诉处理
- 业务交流与知识库
- 系统管理与运维

### 1.3 系统用户

| 用户角色 | 描述 | 权限范围 |
|---------|------|---------|
| 系统管理员 | 负责系统配置、用户管理、权限分配 | 系统管理、全部业务数据 |
| 采购单位用户 | 提报采购计划、发起采购需求 | 本单位数据操作 |
| 采购工作人员 | 编制计划、处理需求、生成合同 | 业务处理全流程 |
| 评审专家 | 进行采购评审、打分 | 评审任务 |
| 财务主管 | 预算审批、财务审核 | 财务相关审批 |
| 部门主管 | 本部门采购审批 | 本部门审批 |

## 6. 业务流程图需求

### 6.1 采购计划流程

```
[年度计划编制] → [单位填报] → [自动汇总] → [计划审批] → [计划下达]
                                    ↓
                              [双向考核]
                                    ↓
                              [统计分析]
```

### 6.2 采购需求流程

```
[需求发起] → [合规检查] → [部门审批] → [财务审批] → [采购审批] → [需求汇总]
                  ↓              ↓              ↓
              [整改任务]     [驳回修改]     [最终确认]
```

### 6.3 采购文件流程

```
[文件创建] → [模板应用] → [内容编辑] → [文件审核] → [审核流转] → [文件归档]
                                  ↓
                            [版本对比]
```

### 6.4 采购评审流程

```
[评审准备] → [材料上传] → [客观打分] → [专家打分] → [结果汇总] → [结果发布]
                                    ↓
                              [异常处理]
```

### 6.5 合同管理流程

```
[合同生成] → [信息抓取] → [合规校验] → [合同签署] → [履约跟踪] → [合同归档]
                                      ↓
                                [节点报警]
```
