## 1. 系统概述

### 1.1 项目背景与目标

智能化采购系统旨在通过信息化手段实现采购全流程的线上化、规范化管理。系统以采购计划为起点，贯穿采购需求、采购文件、采购评审、采购合同、合同履约等关键环节，实现采购业务的全程可追溯、可监控、可分析。

系统核心目标：
- 实现采购计划的线上化、模板化编制与汇总
- 支持采购需求的发起、审核、流转全流程管理
- 提供标准化的采购文件模板与智能辅助编制
- 实现评审过程的线上化管理与智能打分
- 支持合同的智能生成与履约监控
- 提供完整的档案管理与数据查询分析能力

### 1.2 系统范围

系统包含以下11个功能模块：

| 序号 | 模块名称 | 主要功能概述 |
|------|----------|--------------|
| 1 | 采购计划模块 | 年度/季度计划编制、汇总、考核、统计分析 |
| 2 | 采购需求模块 | 需求发起、合规检查、审核流转、智能推荐 |
| 3 | 采购文件模块 | 文件创建、审核、版本管理、安全控制 |
| 4 | 采购评审模块 | 评审准备、投标审核、结果管理、专家管理 |
| 5 | 采购合同模块 | 合同智能生成、信息抓取、模板管理 |
| 6 | 合同履约模块 | 履约信息管理、节点监控、报警管理 |
| 7 | 档案管理模块 | 自动归档、节点管理、档案检索 |
| 8 | 数据查询模块 | 问答查询、条件查询、文件导出 |
| 9 | 质疑投诉模块 | 信息登记、流程办理、统计分析 |
| 10 | 业务交流模块 | 业务咨询、政策查询、知识库管理 |
| 11 | 系统管理模块 | 组织管理、用户管理、权限配置、日志管理 |

| 角色 | 描述 | 主要权限 |
|------|------|----------|
| 采购单位 | 发起采购需求、填报采购计划、提交投标文件的单位 | 计划填报、需求提报、文件上传、质疑投诉 |
| 审核人员 | 负责审核采购计划、需求、文件的审批人员 | 各业务环节的审核、审批操作 |

#### 2.1.2 用户故事

**US-PP-01: 年度计划编制**
- 角色: 采购单位
- 场景: 年初，各单位需编制年度采购计划
- 需求: 支持多单位按标准化模板完成计划填报，实现创建、修改、删除与查询
- 验收标准: 支持多条件组合与模糊匹配查询

**US-PP-02: 计划汇总与下达**
- 角色: 管理员
- 场景: 年末，汇总各单位年度计划
- 需求: 自动汇总多单位填报数据，生成规范汇总表格
- 验收标准: 支持按采购类型、预算区间等方式统计

**US-PP-03: 滞后计划监控**
- 角色: 管理员/审核人员
- 场景: 监控计划执行进度
- 需求: 自动识别滞后项目，发送报警通知
- 验收标准: 支持多渠道消息提醒，分级报警

#### 2.1.3 输入输出定义

**年度计划输入**:
- 计划名称、计划类型、采购预算、预算区间
- 采购单位、编制人、编制时间
- 关联项目信息、历史计划ID

**年度计划输出**:
- 计划汇总表、执行进度表、滞后分析表
- Excel/PDF格式报表

#### 2.1.4 业务流程

```
计划编制 -> 计划提交 -> 单位审核 -> 计划汇总 -> 计划下达 -> 执行跟踪 -> 归档管理
```

## 5. 非功能需求

### 5.1 性能要求

| 指标项 | 要求 |
|--------|------|
| 系统响应时间 | 普通操作 ≤ 2秒，复杂查询 ≤ 5秒 |
| 并发用户数 | 支持至少500用户同时在线 |
| 数据处理能力 | 单日处理业务量 ≥ 10,000笔 |
| 报表生成时间 | 常规报表 ≤ 30秒，复杂报表 ≤ 60秒 |
| 文件上传 | 单文件最大支持100MB |

### 5.2 安全要求

| 安全项 | 要求 |
|--------|------|
| 身份认证 | 支持用户名密码、数字证书等多因素认证 |
| 访问控制 | 基于RBAC的权限管理，功能权限与数据权限分离 |
| 数据加密 | 传输使用HTTPS，存储使用AES-256加密 |
| 安全审计 | 记录所有关键操作日志，支持追溯 |
| 漏洞防护 | 防止SQL注入、XSS、CSRF等常见攻击 |
| 敏感数据 | 身份证号、银行卡号等敏感信息脱敏显示 |
| 密码策略 | 至少8位，包含大小写字母、数字、特殊字符 |

### 5.3 可用性要求

| 指标项 | 要求 |
|--------|------|
| 系统可用性 | ≥ 99.5% |
| 故障恢复时间 | ≤ 30分钟 |
| 数据备份 | 每日增量备份，每周全量备份 |
| 灾难恢复 | 支持异地容灾，RPO ≤ 1小时，RTO ≤ 4小时 |
| 浏览器支持 | Chrome、Firefox、Safari、Edge最新两个版本 |

### 5.4 兼容性要求

| 类型 | 要求 |
|------|------|
| 操作系统 | Windows 10/11、macOS、iOS、Android |
| 终端适配 | 支持PC端、移动端、平板端 |
| 第三方集成 | 支持与财务系统、档案系统的数据对接 |

### 5.5 易用性要求

| 指标项 | 要求 |
|--------|------|
| 界面设计 | 符合采购业务习惯，布局合理，操作简便 |
| 帮助文档 | 提供完整操作手册和在线帮助 |
| 错误提示 | 清晰明确，指导用户正确操作 |
